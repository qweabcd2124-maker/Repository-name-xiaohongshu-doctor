## 二、展评策略（10:00–15:30，核心战场）

### 2.1 评委画像与应对

30-40 名评委，构成：**产品人 + 技术人 + 媒体人 + 投资人**。每位看约 7-8 个项目。

| 评委类型 | 关注点 | 小红薯医生 应对 |
|----------|--------|------------|
| **产品人** | 用户体验、痛点真实性 | 演示完整流程，强调"8000万创作者的真实痛点" |
| **技术人** | 架构深度、创新性 | 展示多 Agent 辩论架构 + 双轨数据分析（传统统计 + LLM） |
| **媒体人** | 故事性、传播性 | "全场唯一中学生队" + 诊断卡片天然是小红书内容 |
| **投资人** | 商业潜力、市场规模 | "小红书 8000 万创作者 × SaaS 订阅" |

### 2.2 三套话术（必须提前练熟）

**30 秒版（评委路过时）：**

> "我们做了一个小红书笔记诊断工具——你发了笔记没人看，不知道问题在哪。小红薯医生用 5 个 AI Agent 辩论 + 真实数据对比，告诉你具体哪里有问题、怎么改。来，把你最近发的笔记截图给我，现场诊断给你看。"

**3 分钟版（评委停下来了）：**

```
[0:00-0:30] Hook + 现场演示启动
  "发小红书最痛苦的不是写笔记，是写完没人看还不知道为什么。
   来，我现在给你看——"
  → 打开 xiaohongshu-doctor.muran.tech，上传一张截图

[0:30-1:30] 等待诊断时讲技术
  "我们不是一个 AI 随便给建议。后面跑着 5 个 Agent——
   内容分析师看你的文案、视觉诊断师看你的封面、增长策略师看标签和发布时间、
   用户模拟器预测评论区反应——它们之间会互相辩论，
   最后由裁判综合出报告。"

[1:30-2:30] 展示报告
  → 报告出来了，展示分数、雷达图、维度条
  "看，你的标题得分偏低——因为美食品类爆款标题平均 18 个字，
   你只写了 8 个。这不是 AI 拍脑袋说的，
   是我们基于 500+ 条真实小红书数据分析出来的基线。"

[2:30-3:00] 收尾 + 引导提问
  "我们还能模拟评论区反应、一键生成优化版标题。
   有什么想看的？"
```

**技术深挖版（评委追问时）：**

- 多 Agent 编排：asyncio.gather 并行 Round 1 → 辩论 Round 2 → Judge Round 3
- 双轨数据分析：传统统计（Spearman 相关性、多元回归、K-Means 聚类）+ LLM 深度分析（mimo-v2-omni 封面视觉理解、mimo-v2-pro 内容模式总结）
- 量化基线：8 品类 × 23 项统计指标，含粉丝分层、标签分桶 vs 互动率
- 3 档模型调度：flash（快速评论生成）、pro（1M 上下文诊断）、omni（多模态封面/视频分析）

## 三、五维评分逐项拆解 × 小红薯医生 对策

### 维度 1：创意（方向已定，重构叙事）

**小红薯医生 的创意壁垒不在"AI诊断笔记"这个概念（太宽泛），在于三层差异化：**

1. **多 Agent 辩论**（不是一个 AI 说了算）— 展示 Agent 之间观点冲突的时间线
2. **真实数据基线**（不是 AI 拍脑袋）— "美食品类爆款标题平均 18 个字"这种可验证的结论
3. **评论区模拟**（预测而不仅是分析）— 这是别人没做的功能

**PPT 叙事弧：**
```
"小红书 8000 万创作者 → 90% 不知道笔记为什么没火
  → 现有工具都是 AI 凭感觉给建议 → 我们用数据 + 多 Agent 辩论
  → 不是告诉你'标题要有吸引力'，而是告诉你'美食品类爆款标题 16-20 字，你的只有 8 个'"
```

### 维度 2：现场讲述（最可控的杠杆）

**核心原则：让评委参与，不是让评委听。**

- 现场打开 xiaohongshu-doctor.muran.tech 让评委自己上传截图 → 体验比讲解强 10 倍
- 如果网络不稳，立刻切到预跑好的 3 个离线 fallback 案例
- 绝不超时。展台 3 分钟讲完，决赛 6 分钟讲完。剩余时间让评委问

**年龄身份策略：**
- 不要主动说"我们才 13 岁"（显得要同情分）
- 先展示产品实力，让评委主动发现年龄 → 惊讶效果 >>直接说
- 但在队伍介绍材料里写清楚，评委看资料时会看到

### 维度 4：商业潜力（一页 PPT 讲清楚）

```
市场：小红书 8000 万创作者
痛点：不知道笔记为什么没火，没有系统性诊断工具
商业模式：
  - 免费版：每月 3 次诊断
  - Pro 版：¥29/月，无限诊断 + 历史对比 + 批量分析
  - MCN 版：按席位收费，团队管理 + API 接入
竞品：无直接竞品（灰豚/新红仅做数据监控，不做诊断+优化）
壁垒：数据飞轮——用户越多，诊断越准，竞品无法复制积累的数据
```

### 维度 5：技术难度

**小红薯医生 的技术栈深度远超普通黑客松项目：**

| 层次 | 技术 | 复杂度 |
|------|------|--------|
| 多 Agent 系统 | 5 Agent 并行+辩论+裁判，asyncio 编排 | 高 |
| 三档模型调度 | flash/pro/omni 按任务智能分配 | 中高 |
| 多模态分析 | OpenCV 图像特征 + LLM 视觉理解 + 视频分析 | 高 |
| 量化研究系统 | Spearman/回归/KMeans/PCA + LLM 双轨 | 高 |
| 数据工程 | SQLite baseline + 实时对比 + 统计指标 | 中 |
| 全栈部署 | React+FastAPI+nginx+SSL+systemd | 中 |

**评委问"你们怎么做的"时的回答模板：**
> "我们用了三层架构：底层是真实数据分析，中层是 5 个 AI Agent 并行诊断加辩论，顶层是量化评分系统。每个 Agent 看不同维度——内容、视觉、增长策略、用户反应——然后互相辩论，最后裁判综合。不是一个 AI 拍脑袋给建议。"

> **"小红薯医生——用真实数据 + 多 Agent 辩论，给你的小红书笔记开处方。"**
