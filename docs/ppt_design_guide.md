# 小红薯医生 — PPT 设计指南

## 一、硬性约束

| 规则 | 说明 |
|------|------|
| 模板 | 以官方模板为主，**仅 2-3 页**可用自定义背景 |
| 比例 | 宽屏 16:9（舞台主屏为超宽屏） |
| 禁止内容 | 政治人物、争议性公众人物、非官方二维码 |
| 允许 | 小红书二维码、简单动效 |
| 提词器 | 使用 PPT 备注功能，脚下有提词屏 |
| 页数 | 不限，但自行控制时长（展评 3min / 决赛 6min） |

---

## 二、认知科学原则

### 每页只传递一个信息

评委一天看几十个项目，认知资源严重枯竭。

```
❌ 一页塞满：标题 + 架构图 + 数据表 + 三段文字
✅ 三层结构：大标题 → 关键数字/截图 → 一句话解释
```

### 首因效应：第一页定生死

```
❌ 第一页："PageOne 队伍介绍 — 姜睦然、陈宇夏..."
✅ 第一页：产品界面全屏截图 + 一句话
   "你的笔记为什么没火？小红薯医生 30 秒告诉你答案。"
```

评委对你的第一印象在前 10 秒锁定。

### 峰终定律：最后一页锁记忆

```
❌ 最后一页："谢谢！Q&A"
✅ 最后一页：
   "8000 万创作者，每一篇笔记都值得被看见。"
   小红薯医生 · xiaohongshu-doctor.muran.tech
```

### 锚定效应：先说数字

```
❌ "我们的系统分析速度很快"
✅ "30 秒，5 个 AI Agent 完成一次完整诊断"
❌ "我们分析了很多真实数据"
✅ "501 条真实笔记 × 23 项统计指标 × 8 个品类"
```

## 三、页面结构（建议 10-12 页）

### 展评版（3 分钟，7-8 页精简）

| 页码 | 内容 | 模板/自定义 | 设计要点 |
|------|------|------------|---------|
| 1 | **Hook 页**：产品截图 + 一句话定位 | 自定义 | 全屏产品截图，文字叠加半透明底 |
| 2 | **痛点**：8000 万创作者的困境 | 官方模板 | 一个数字 + 一句痛点描述 |
| 3 | **产品演示**：核心流程截图 | 自定义 | 3-4 张关键界面截图拼接 |
| 4 | **技术架构**：多 Agent 辩论图 | 官方模板 | 一张架构图，不要文字堆砌 |
| 5 | **数据基线**：量化研究成果 | 官方模板 | 1-2 张统计图表（相关性热力图/回归系数） |
| 6 | **差异化**：为什么我们不一样 | 官方模板 | 三个对比点，每个一行 |
| 7 | **商业模式** | 官方模板 | 市场规模 + SaaS 定价，一页讲清 |
| 8 | **收尾**：金句 + 团队 | 自定义 | 队伍照片 + 收尾句 |

### 决赛版（6 分钟，扩展到 10-12 页）

在展评版基础上插入：

| 插入位置 | 额外页 | 内容 |
|----------|--------|------|
| 页 3 后 | 数据验证页 | 用已知爆款反向验证诊断准确性 |
| 页 4 后 | 研究方法页 | 双轨分析：传统统计 + LLM 深度分析 |
| 页 5 后 | 用户画像页 | 评论区模拟的数据基础 |
| 页 7 后 | 自进化系统页 | 数据飞轮：越用越准 |

## 四、自定义页面设计规范（2-3 页）

### 配色

| 元素 | 颜色 | 用途 |
|------|------|------|
| 主色 | `#ff2442` 小红书红 | 标题、按钮、强调 |
| 文字 | `#262626` | 正文 |
| 辅助 | `#999999` | 说明文字 |
| 背景 | `#fafafa` 或 `#ffffff` | 干净、不抢注意力 |

### 字体

- 标题：思源黑体 Bold 或 PingFang SC Semibold，36-44pt
- 正文：思源黑体 Regular，20-24pt
- 数据：Inter 或 DIN，数字要突出

### 截图处理

- 产品截图加圆角 + 轻微阴影（模拟真实设备）
- 如果放手机截图，用 mockup 框架
- 截图内容要清晰可读（放大关键区域）

## 五、数据可视化页面

展示研究成果的页面是核心差异化。建议放：

1. **相关性热力图**（`data/research_output/charts/correlation_heatmap.png`）
   - 说明："我们分析了 500+ 条真实笔记的 12 个变量间的相关性"

2. **回归系数对比**（`data/research_output/charts/regression_coefficients.png`）
   - 说明："控制粉丝量后，不同因素对互动量的独立影响"

3. **品类雷达图**（`data/research_output/charts/category_radar.png`）
   - 说明："每个品类的爆款参数都不一样"

这些图表是**我们的技术护城河视觉化** — 别的队没有这个。

## 六、演讲备注模板（提词器用）

每页 PPT 的备注写成这样：

```
【页面标题】产品演示

【说什么 - 60 字以内】
"来看小红薯医生怎么工作。我上传一张美食笔记的截图——
30 秒后，5 个 Agent 给出了完整诊断。
看，标题得分只有 52 分，因为美食爆款标题平均 18 个字，
这篇只写了 8 个。"

【如果时间紧】跳过细节，直接翻到报告截图
【如果评委追问】展开讲 Agent 辩论时间线
```
